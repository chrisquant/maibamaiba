## 1. 当前工程对齐基线（必须遵守）

### 1.1 技术栈与依赖基线

- Expo SDK 54（托管 RN 版本，不手动锁定 RN 小版本）
- React Navigation：`@react-navigation/native` + `@react-navigation/bottom-tabs` + `@react-navigation/native-stack`
- 数据层：`expo-sqlite`
- 日期选择：`@react-native-community/datetimepicker`
- 图表绘制：`react-native-svg` + 自研 `LineChart` 组件（不依赖 `victory-native` 的旧 API）

### 1.2 Babel 基线（关键）

- `babel.config.js` 必须存在并包含：
  - `presets: ["babel-preset-expo"]`
  - `plugins: ["react-native-reanimated/plugin"]`
- `package.json` 必须包含 `babel-preset-expo`，否则会出现：
  - `Cannot find module 'babel-preset-expo'`

### 1.3 启动与清缓存规范

- 依赖变更后统一使用：
  - `npx expo start -c`
- 如出现“看起来像旧报错”的情况，优先判断 Metro 缓存而不是代码回退。

## 3. 问题复盘与强制修正规范

## 3.1 iOS SafeArea 覆盖问题

### 问题
- 首页顶部内容压住系统状态栏（时间/WiFi/灵动岛区域）。

### 规范
- 所有 Tab 主页面必须使用 `SafeAreaView` 且 `edges={["top"]}`。
- 顶部容器禁止硬编码贴顶布局。

## 3.2 表单键盘导致无法点击保存

### 问题
- 新增资产页键盘弹起后无法收起，底部按钮不可点。

### 规范
- 表单页必须同时使用：
  - `KeyboardAvoidingView`
  - `keyboardShouldPersistTaps="handled"`
  - `keyboardDismissMode="on-drag"`
  - 点击空白 `Keyboard.dismiss()`

## 3.3 iOS 日期选择器“滑动即提交”

### 问题
- iOS 选择日期时，用户还没确认就自动收起并写值，体验差。

### 规范
- iOS 日期选择采用“临时值 + 确认提交”模式：
  - 滑动仅更新临时变量
  - 点击“确认”后写入字段
  - 点击“取消”不改原值
- Android 可保留系统默认确认行为。

## 3.4 折线图时间序列不连续

### 问题
- 仅在“有资产变更日期”打点，导致时间轴断裂，不是完整序列。

### 规范
- 趋势图必须生成完整时间序列：
  - 起点：当前范围起点（或最早购入日）
  - 终点：今天
  - 粒度：日 / 周 / 月
- 即使某时间点无新增资产，也必须有数据点。

## 3.5 趋势图跨度过长，难以查看全貌

### 问题
- 按天全量渲染造成图过长，用户需长距离横滑。

### 规范
- 默认使用完整时间序列数据，不允许为“适配一屏展示”而强制抽样压缩。
- 通过手势改变可视窗口（平移与缩放），而不是丢弃数据点。
- 如未来需性能兜底，降采样必须做成可配置开关，且默认关闭。

## 3.6 统计页滚动冲突

### 问题
- 上部图表和下部列表嵌套滚动，无法顺畅向下查看资产详情列表。

### 规范
- 统计页采用单一纵向滚动容器：
  - 推荐 `FlatList + ListHeaderComponent`
  - 禁止 `ScrollView` 嵌套 `FlatList` 的双滚动冲突结构

## 3.7 图表节点 key 重复

### 问题
- `Encountered two children with the same key ...` 导致渲染异常。

### 规范
- SVG 点和标签 key 必须保证唯一（优先索引或稳定唯一 id）。
- 不可仅使用可能重复的日期文本作为 key。

## 3.8 图表右边界裁切

### 问题
- 右侧最后日期/数值被裁掉。

### 规范
- 折线图右侧必须预留足够 padding。
- 首尾标签使用 `start/end` 对齐策略，避免文本超边界。

## 4. 统计页标准实现规则（修订后）

### 4.1 时间范围与粒度

- 范围：全部 / 本周 / 本月 / 本年
- 粒度：按天 / 按周 / 按月
- 双指缩放规则：
  - 仅改变时间轴可视窗口宽度（比例尺）
  - 不自动切换数据点粒度
  - 粒度仅允许用户手动切换（按天 / 按周 / 按月）

### 4.2 趋势值计算口径

- 资产总额趋势：`price + addFee` 的累计视图（按时间点）
- 日均成本趋势：按时间点重算平均成本，不仅使用“购买日快照”
- 非服役资产按状态规则冻结或限制累计窗口（与数据库规则一致）

### 4.3 视觉与交互

- 默认应“一眼可看首尾”
- 支持单指左右拖拽平移时间窗口
- 支持双指捏合缩放时间轴比例尺（保持手势中心时间点稳定）
- 下方资产排序列表可持续向下滚动

### 4.4 图表手势落地规则（新增）

- 单指横向拖拽：
  - 修改 `visibleStart / visibleEnd`（或等价的 start/count）实现时间窗口平移
  - 触达数据边界后停止平移，不得进入无数据区
- 双指捏合缩放：
  - 仅修改可视窗口宽度，不改时间粒度
  - 以双指中心对应时间点为锚点重算窗口，避免缩放跳跃
  - 需限制最小/最大窗口宽度（最小显示点数、最大为全量）
- 手势冲突规则：
  - 双指手势优先识别为缩放，不触发单指平移
  - 单指横向拖拽优先控制图表平移，纵向滚动交给页面列表
- 验收标准：
  - 缩放后粒度标签不变化（除非用户手动点击切换）
  - 图表展示点位来自完整序列窗口截取，不得因默认采样丢点
  - 平移与缩放过程无明显抖动、无越界、无手势卡死

## 5. 资产新增页标准实现规则（修订后）

### 5.1 输入边界

- 购买日期不可晚于今天
- 使用次数必须为 `>= 0` 整数
- 卖出残值必须为 `>= 0`
- 附加费允许负值，但总费用不可为负

### 5.2 键盘与日期交互

- 键盘弹起后仍可点击保存
- iOS 日期必须“先选后确认”
